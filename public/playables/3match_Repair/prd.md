# Product Requirements Document (PRD)

## 1. 프로젝트 개요 (Project Overview)
- **프로젝트 명:** Spooky Hotel Match & Repair (가칭)
- **장르:** 매치 3 퍼즐 (Match-3 Puzzle) + 인테리어 복구 (Repair & Decoration)
- **지원 플랫폼:** PC, 모바일 (웹 브라우저 및 클라이언트 지원)
- **화면 규격:** 스마트폰 세로 비율 (Portrait Mode, 9:16) 최적화

## 2. 게임 코어 루프 (상세 작동 방식)
게임은 다음의 흐름으로 진행되며, 모든 수리 대상(퀘스트)에 대해 액션을 마친 후 **최종 별점을 누적하여 정산**하는 방식으로 진행됩니다.

1. **메인 화면 (Spooky Hotel):**
   - 게임 시작 시 다소 어둡고 으스스한(spooky) 분위기의 망가진 호텔 실내가 화면에 등장합니다.
   - 방 안의 크게 망가진 구역 4곳(예: 부서진 침대, 테이블 위 망가진 축음기, 찢어진 소파, 깨진 창문)의 옆에 **퀘스트 말풍선**이 나타납니다.
2. **퍼즐 진입:**
   - 화면에 떠 있는 말풍선을 클릭하면 해당 구역 수리에 필요한 아이템을 획득하기 위한 **매치 3 (3-match) 인게임 화면**으로 진입합니다.
3. **매치 3 인게임 진행 및 결과 판정:**
   - 제한된 `Move(턴)` 내에 상단에 위치한 목표 미션 블록을 제거해야 합니다.
   - **성공 시:** "고급 보상 아이템(예: 최고급 침구류)"을 획득하며 아웃게임으로 복귀하고, 선택했던 퀘스트 말풍선은 사라집니다. 획득한 보상은 화면 하단 인벤토리에 들어옵니다.
   - **실패 시:** 턴을 모두 소모할 경우 아웃게임으로 곧바로 복귀합니다. 선택했던 퀘스트 말풍선은 사라지지만, **보상은 획득하지 못한 채** 빈 손으로 메인 화면에 남게 됩니다.
4. **수리 및 교체 인터랙션 (드래그 판정):**
   - 유저는 인벤토리에 들어온 아이템을 터치하여 화면상의 망가진 위치로 직접 드래그 & 드롭(Drag & Drop)합니다.
   - **올바른 위치 (성공):** 대상 오브젝트가 사르륵 교체되는 애니메이션이 재생되며 성공적으로 수리 완료 처리됩니다. 
   - **올바르지 않은 위치 (실패):** 
     - 드롭 위치가 오답일 경우, **우스꽝스럽게 상황을 망치는 애니메이션(예: 고급 침구류를 깨진 창문에 드롭 시 창문 밖으로 돌돌 말아 세게 던져버리는 연출)**이 재생됩니다.
     - 애니메이션이 끝난 직후 해당 아이템은 튕겨 되돌아오는 것 없이 **완전히 소멸**하며, 이로써 해당 수리 기회는 날아가버린 것으로 간주됩니다.
5. **최종 정산 및 클리어 판정 (5성급 호텔 달성도):**
   - 4개의 퀘스트 말풍선이 화면상에서 모두 제거되면 "최종 결과 정산 화면"이 팝업됩니다.
   - 초기 화면에서는 5개의 빈 별점 칸(☆☆☆☆☆)이 제시됩니다.
   - **별점(★) 획득 공식 (최대 5개):**
     - **고정 점수:** 4개의 말풍선 퀘스트를 모두 소모하여 게임의 끝까지 도달했을 시 -> **★ 1개 획득**
     - **추가 점수:** 각 인테리어 요소(침대, 축음기 등)를 올바른 아이템 드래그로 "성공적으로 고쳐냈을 때마다" -> **★ 1개씩 추가 획득** (총 4개)
   - **승패 판정:**
     - **3성 이상(★★★~★★★★★) 달성 시:** Game Clear. 호텔 내부 조명이 화사하게 밝아지고 멋지게 복구된 인테리어 완성 이미지가 출력됩니다.
     - **3성 미만(★★ 이하) 달성 시:** Game Fail. 최악의 분위기와 엉망인 인테리어가 그대로 유지되는(또는 더 우스꽝스러워진) 화면을 보여주며 게임에 실패합니다.

## 3. 기능 개발 명세 상세 (Detailed Features)

### 3.1 호텔 메인 뷰 (Lobby & Map)
- **상태 관리 체계 (State Management):**
  - 말풍선 노출 여부 관리: 총 4개, 퀘스트 플레이 시 성공/실패 여부와 관계없이 소멸 트리거 작동.
  - 드롭 오답 연출: 특정 오답 짝꿍에 맞춰서, 아이템을 없애버리면서 코믹한 애니메이션 이펙트를 보여줄 수 있는 별도 UI 처리.

### 3.2 매치 3 퍼즐 코어 및 조합 로직 (Match-3 System)
- **이동(Move) 판정 및 턴 차감 방식:**
  - 유저가 인접한 두 블록을 터치 및 스와이프하여 자리를 교환합니다.
  - **매치 조건 미충족 시:** 자리 교환 후 가로 또는 세로로 3개 이상의 동일 색상 블록이 연결되지 않으면, 퍼즐 파괴 트리거나 콤보가 작동하지 않고 **두 블록이 다시 원래 자리로 되돌아가는(Bounce Back) 애니메이션**이 재생되며 **이동 횟수(Move)는 차감되지 않습니다.**
  - **매치 조건 충족 시:** 교환 결과 블록이 3개 이상 성공적으로 매칭되어 파괴 애니메이션(콤보 체인 포함)이 발동된 직후에 비로소 **Move가 1 차감**됩니다.
  
  - **마지막 이동(Last Move) 및 승패 평가지연(Zero-Move Check) 로직:**
    - 남은 턴(Move)이 1인 상태에서 유저가 마지막 블록 스와이프를 진행할 경우 **Move가 0이 되더라도 즉시 미션 실패(Fail)를 출력하지 않습니다.**
    - 스와이프로 인해 발생하는 모든 **특수 블록 폭발, 연쇄 매치(Cascade Combo), 빈 공백으로 블록이 낙하하며 추가로 터지는 파괴 연출이 모두 완전히 정지하여 보드판에 더 이상 액션이 없을 때까지 승패 평가는 대기**합니다.
    - 보드 내 모든 연쇄 작용이 정지한 최후의 시점에 미션 목표를 검사합니다. 마지막 연쇄 폭발을 통해 목표를 충족했다면 Move가 0이더라도 **Mission Clear(성공)**로 보상을 획득하며, 모든 연쇄가 끝난 시점에도 목표량에 도달하지 못했을 때만 최종적으로 Mission Fail로 확정됩니다.
  
- **특수 블록 생성 및 조합 로직 (Combinations):**
  일반적인 장르 표준 매칭 조합 공식을 모두 지원하여 역동적인 퍼즐 플레이를 유도합니다.
  - **4개 직렬 매칭 (가로/세로 4-Match):** 
    - 연결된 모양에 따라 가로 또는 세로 모양 무늬가 들어간 **'라인 블록(Line Bomb)'**이 생성됩니다. 
    - 작동 방식: 터치 또는 다른 블록과 매칭 시 해당 가로 한 줄 또는 세로 한 줄을 전부 파괴합니다.
  - **L자 또는 T자 형태 매칭 (5개 이상 교차):** 
    - 강력한 구역 파괴용인 **'폭탄 블록(Bomb)'**이 생성됩니다.
    - 작동 방식: 주변 3x3 반경 또는 십자 모양 반경 내의 모든 일반 블록을 기폭시킵니다.
  - **5개 직렬 매칭 (5-Match):**
    - 화면 전체를 제어할 수 있는 가장 강력한 **'컬러 미러볼 블록(Color Bomb)'**이 생성됩니다.
    - 작동 방식: 대상 색상의 일반 블록과 스와이프 할 경우 화면에 존재하는 동일 색상의 보드 판 블록을 모두 일제히 타격 및 제거합니다.
  - **특수 블록 간의 연계 교차 기믹 (Combo Chain):**
    - `라인 블록` + `폭탄 블록`: 상하좌우로 3줄 굵기의 거대한 십자 크로스를 그리며 모두 파괴.
    - `컬러 미러볼` + `일반 특수 블록(라인 또는 폭탄)`: 보드판 내에 특유의 색상과 동일한 모든 일반 블록을 특수 블록으로 강제 복제&변환시킨 후, 전체 화면을 연쇄 기폭시켜 화려하게 화면을 씻어냅니다.

### 3.3 인벤토리 UI & Drag & Drop 인터랙션
- **완전 소멸 방식 적용:** 올바른 타겟, 오답 타겟 모두 드롭 존에 닿아 이벤트가 발동되는 즉시 인벤토리 상태 배열에서 해당 아이템을 제거/삭제(pop).

## 4. 기술 및 개발 환경 (Technical Stack)
- **상태 관리 체계 (Store System):**
  - 퍼즐 엔진 상태: 터치/스와이프 한 두 블록의 ID 값을 추적하고 매치 판정 함수 실행, 통과시에만 Move 차감 및 콤보 로직 작동 연동.
  - `questsRemaining`: 4에서 시작하여 3-match가 끝날 때마다 -1 연산, 0 도달 시 최종 정산(Result Screen) 트리거.
  - `successfulRepairs`: 수리가 성공적으로 끝날 때마다 +1 반영, 최종 Result Screen에서 `1(기본) + successfulRepairs`로 총 획득한 Star 계산.
- **에셋 및 리소스 교체 용이성 (Resource Replacement Structure):**
  - **더미 리소스 최우선 사용:** 초기 프로토타이핑 및 코딩 시 시각적 흐름 테스트를 위해 게임 내 모든 오브젝트(블록, 침대 조각, 배경 등)는 단순화된 임시 더미 이미지를 생성하여 배치합니다.
  - **리소스 경로 모듈화:** 추후 작업자가 직접 디자인한 리소스를 손쉽게 대체할 수 있도록 모든 이미지의 파일 경로 관리는 별도의 상수 파일(예: `assets.config.ts`)이나 단일화된 디렉터리 세팅(예: `public/assets/blocks/...`)으로 분리 설계합니다.
  - **하드코딩 방지:** 컴포넌트 깊숙한 곳에 이미지 파일명을 하드코딩하지 않고, 언제든지 `block_red.png` 같은 원소스를 덮어쓰거나 경로를 수정할 수 있게 유연성을 보장합니다.
